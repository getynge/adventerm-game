## cbindgen config

`adventerm_ffi/cbindgen.toml`:

```toml
language = "C"
header = "/* AUTO-GENERATED by build.rs from src/. Do not edit by hand. */"
include_guard = "ADVENTERM_FFI_H"
documentation = true
documentation_style = "doxy"

[export]
# We already prefix exports manually (game_, battle_, console_, save_, ffi_, item_kind_, etc.)
prefix = ""

[enum]
# Emits e.g. CDirection_Up rather than just Up at the C level — disambiguates
# variant names across enums.
prefix_with_name = true
rename_variants = "ScreamingSnakeCase"

[parse]
# We do NOT introspect adventerm_lib. Only the FFI crate's own #[repr(C)]
# types and extern "C" functions appear in the header. This is essential to
# the Rust-first constraint.
parse_deps = false

[fn]
# Sort functions by name in the header, for stable diffs.
sort_by = "Name"

[struct]
derive_eq = false
derive_neq = false
```

Header lives at `adventerm_ffi/include/adventerm_ffi.h` and is **committed to the repo** so consumers (Swift Package, CI, smoke tests) get a stable artifact without running cbindgen.

## CI: regeneration is a no-op

```bash
cargo build -p adventerm_ffi
git diff --exit-code adventerm_ffi/include/
```

Fails the PR if the committed header is stale. Forces contributors to re-run `cargo build` and commit the regenerated header alongside their FFI changes.

## iOS XCFramework

`adventerm_ffi/scripts/build-xcframework.sh`:

```bash
#!/usr/bin/env bash
set -euo pipefail

CRATE_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"
WORKSPACE_DIR="$(cd "${CRATE_DIR}/.." && pwd)"
TARGET_DIR="${WORKSPACE_DIR}/target"
OUT_DIR="${CRATE_DIR}/build/AdventermFFI.xcframework"

# Build all four iOS-family targets in release.
for target in aarch64-apple-ios aarch64-apple-ios-sim x86_64-apple-ios; do
    rustup target add "$target" >/dev/null
    cargo build --release -p adventerm_ffi --target "$target"
done

# Fat archive for the simulator slice (arm64 + x86_64).
mkdir -p "${TARGET_DIR}/ios-sim-fat/release"
lipo -create \
    "${TARGET_DIR}/aarch64-apple-ios-sim/release/libadventerm_ffi.a" \
    "${TARGET_DIR}/x86_64-apple-ios/release/libadventerm_ffi.a" \
    -output "${TARGET_DIR}/ios-sim-fat/release/libadventerm_ffi.a"

# Bundle the XCFramework.
rm -rf "$OUT_DIR"
xcodebuild -create-xcframework \
    -library "${TARGET_DIR}/aarch64-apple-ios/release/libadventerm_ffi.a" \
    -headers "${CRATE_DIR}/include" \
    -library "${TARGET_DIR}/ios-sim-fat/release/libadventerm_ffi.a" \
    -headers "${CRATE_DIR}/include" \
    -output "$OUT_DIR"

echo "Built $OUT_DIR"
```

Invoked manually by release engineers, or by a release CI job. Not required for per-PR CI (the per-target cargo build steps are sufficient there).

## Consolidated Rust test plan

| Test file | Milestone | Coverage |
|-----------|-----------|----------|
| `tests/handle_lifecycle.rs` | M2 | Alloc/free for every handle type; null-free is a no-op |
| `tests/error_codes.rs` | M2 | Provoke each `FfiError` variant; assert `ffi_last_error_message` returns non-null where expected |
| `tests/enum_round_trip.rs` | M3 | For every `CXxx`, `From<lib>` then `TryFrom<u8>` round-trips for every variant |
| `tests/queries.rs` | M3 | Scalar queries match direct `GameState` reads |
| `tests/action_dispatch.rs` | M4 | Each of the nine action entries matches direct `dispatch::<Action>` outcome |
| `tests/iteration.rs` | M5 | Pattern A, B, C accessors match direct iteration; `BufferTooSmall` returns correct `out_required` |
| `tests/save_round_trip.rs` | M6 | `game → save → bytes → save → game` preserves position, stats, inventory; bad bytes return `SaveFormat`/`UnsupportedSaveVersion` |
| `tests/battle_flow.rs` | M7 | Scripted battle resolves to known outcome on a seeded game; log lines readable; error codes for wrong-turn/empty-slot |
| `tests/console_flow.rs` | M8 | Submit `fullbright` flips `game_fullbright`; unknown command sets `was_error`; completion lists known commands; registry introspection |

All tests link the FFI crate as `rlib` (Rust integration tests, not C smoke).

## Miri scope

Cheap UB detection on lifecycle and pure-conversion suites. Skip iteration/save (too slow under Miri):

```bash
cargo +nightly miri test -p adventerm_ffi \
    --test handle_lifecycle \
    --test error_codes \
    --test enum_round_trip \
    --test queries
```

Run on the Linux CI runner. Catches UAF, out-of-bounds writes via `*out_*` pointers, and provenance bugs in the conversion code.

## C smoke test (M9)

`adventerm_ffi/tests/smoke.c`:

```c
#include <assert.h>
#include <stdio.h>
#include <stdlib.h>
#include <string.h>
#include "adventerm_ffi.h"

int main(void) {
    GameHandle* game = game_new_seeded(42);
    assert(game != NULL);

    size_t x = 0, y = 0;
    int rc = game_player_pos(game, &x, &y);
    assert(rc == 0);
    printf("player at %zu,%zu\n", x, y);

    CMoveOutcome outcome = {0};
    rc = game_action_move(game, /* CDirection_Up */ 0, &outcome);
    assert(rc == 0);
    printf("move outcome tag=%u\n", outcome.tag);

    size_t inv_len = 0;
    rc = game_inventory_len(game, &inv_len);
    assert(rc == 0);

    SaveHandle* save = NULL;
    rc = save_new_from_game(game, "smoke", &save);
    assert(rc == 0);
    assert(save != NULL);

    size_t needed = 0;
    rc = save_to_bytes(save, NULL, 0, &needed);
    assert(rc == /* BufferTooSmall */ -12);
    assert(needed > 0);

    uint8_t* buf = malloc(needed);
    rc = save_to_bytes(save, buf, needed, &needed);
    assert(rc == 0);

    SaveHandle* restored = NULL;
    rc = save_from_bytes(buf, needed, &restored);
    assert(rc == 0);

    GameHandle* restored_game = NULL;
    rc = save_to_game(restored, &restored_game);
    assert(rc == 0);

    size_t rx = 0, ry = 0;
    game_player_pos(restored_game, &rx, &ry);
    assert(rx == x && ry == y);

    free(buf);
    save_free(save);
    save_free(restored);
    game_free(restored_game);
    game_free(game);

    printf("smoke OK\n");
    return 0;
}
```

CI step (Linux):

```bash
cargo build --release -p adventerm_ffi
cc -Wall -Wpedantic \
   adventerm_ffi/tests/smoke.c \
   -L target/release \
   -ladventerm_ffi \
   -ldl -lpthread -lm \
   -Iadventerm_ffi/include \
   -o /tmp/smoke
/tmp/smoke
```

## Header lint

In CI, additionally syntax-check the header standalone:

```bash
clang -Wall -Wpedantic -fsyntax-only -x c \
    -Iadventerm_ffi/include adventerm_ffi/include/adventerm_ffi.h
```

Catches any cbindgen output that isn't valid standalone C.

## `agents/ffi.md` (M10)

New file at [agents/ffi.md](../../agents/ffi.md). Covers:

1. **Crate location:** `adventerm_ffi/`. Crate-types and what each is for.
2. **Handle conventions:** alloc/free pairs, null-safe free, no parent/child borrows, single-threaded per handle.
3. **Error codes:** the `FfiError` table, the `LAST_ERROR` mechanism, how to read `ffi_last_error_message`.
4. **Header location and regeneration:** `adventerm_ffi/include/adventerm_ffi.h`, generated by `build.rs`, regenerate via `cargo build -p adventerm_ffi`. CI guarantees no diff after build.
5. **Per-platform build commands:** link to `scripts/build-xcframework.sh` for iOS; `cargo build --release --target <T>` for desktop/Android.
6. **Swift consumer guide:**
   - Wrap a handle in a Swift class with `deinit { game_free(handle) }`
   - Two-call buffer pattern with a Swift extension helper
   - Map `FfiError` rawValue → Swift `enum: Error`
7. **What does NOT cross the FFI boundary:** trait objects, `Vec<T>` directly, `HashMap`, lifetimes, generics, ECS internals (`World`, `ComponentStore`), event bus.
8. **Stability rules:** enum discriminants are wire-stable (APPEND only); function signatures are wire-stable per release; `SAVE_VERSION` bumps invalidate older saves on the FFI side identically to the Rust side.

`agents/README.md` gets one row added to the index pointing to `agents/ffi.md`.

```yaml
          cargo build -p adventerm_ffi --target aarch64-linux-android
          cargo build -p adventerm_ffi --target armv7-linux-androideabi
```
